# 校园集市 — 数据库实验规格说明

## 一、项目概述

校园集市是一个集**信息交流、二手交易、跑腿服务、课程评价、兼职招聘、社交互动**于一体的校园社区平台。

**技术栈：** Python + Flask + pymysql + MySQL 8.0

---

## 二、系统架构

```
浏览器 → Flask Web 应用 → pymysql → MySQL 8.0
           │
           ├── Jinja2 模板渲染
           ├── Session 登录认证
           └── AJAX 异步交互
```

**6 大功能模块：**

| 模块 | 功能 |
|------|------|
| 🏠 主页 | 平台介绍 |
| 📌 最新 | 红包帖 + 最新发布双栏展示 |
| 📂 浏览 | 9 大板块浏览 + 搜索 |
| ✏️ 发帖 | 普通帖 / 红包帖发布 |
| 💬 消息 | 回复我的 / 收到的赞 / 我的点赞 / 我的码住 |
| 👤 我的 | 个人信息、学校、等级、红包币、头像上传 |

## 三、数据库设计

### ER 图核心关系

```
sections ── 1:N ── posts ── 1:N ── comments
           │               ├── 1:N ── likes
           │               ├── 1:N ── favorites
           │               ├── 1:N ── reports
           │               ├── 1:N ── post_views
           │               └── 1:N ── red_distribution
           │
users ──── 1:N ── posts
           │
posts ──── BEFORE DELETE ── post_log (触发器)
```

### 9 张业务表

**users（用户表）** — user_id, nickname, phone, password_hash, avatar, school, role, status, exp, level, red_balance, red_sent, red_received

**sections（板块表）** — sec_id, name, description, sort_order（9个板块：集市24小时火文、红包帖、课程评价、跑腿代办、打听求助、恋爱交友、校园趣事、兼职招聘、二手闲置）

**posts（帖子表）** — post_id, user_id, sec_id, title, content, contact, red_amount, red_remaining, pub_time, status, view_count, like_count, comment_count, hot_score

**comments（评论表）** — cmt_id, post_id, user_id, content, cmt_time

**likes（点赞表）** — post_id, user_id（联合主键）, like_time

**favorites（收藏表）** — post_id, user_id（联合主键）, fav_time

**reports（举报表）** — rp_id, post_id, reporter_id, reason, status, report_time

**post_views（浏览去重表）** — post_id, user_id（联合主键）, view_time

**red_distribution（红包分配表）** — dist_id, post_id, from_user_id, to_user_id, amount, dist_time

### 1 张自动日志表

**post_log** — 由触发器 `trg_post_delete` 自动写入，记录被删帖子的 post_id, title, user_id, deleted_at

## 四、数据库高级特性

### 触发器（5个）

| 触发器 | 表 | 时机 | 功能 |
|--------|-----|------|------|
| trg_after_insert_comment | comments | AFTER INSERT | comment_count+1，重算 hot_score |
| trg_after_delete_comment | comments | AFTER DELETE | comment_count-1，重算 hot_score |
| trg_after_insert_like | likes | AFTER INSERT | like_count+1，重算 hot_score |
| trg_after_delete_like | likes | AFTER DELETE | like_count-1，重算 hot_score |
| trg_post_delete | posts | BEFORE DELETE | 写入 post_log 日志表 |

**热度分公式：** `hot_score = view_count × 0.3 + comment_count × 0.5 + like_count × 0.2`

### 视图 & 存储过程

已按最终精简设计移除，核心逻辑在应用层通过 pymysql 参数化查询实现。

### 事务与并发控制

- MySQL InnoDB 引擎，默认 REPEATABLE READ 隔离级别
- 单条 UPDATE/INSERT 利用 InnoDB 行锁保证原子性
- 红包分配为多步复合操作，设计上可通过显式事务 + SELECT FOR UPDATE 实现更严格的并发控制

## 五、核心功能逻辑

### 1. 用户认证

- 学号 + 密码登录，SHA-256 哈希存密码
- Flask Session 管理登录状态
- 所有页面 `@login_required` 保护

### 2. 发帖

- 选择板块 + 填写标题/内容 → INSERT posts
- 红包帖：检查红包币余额 → 扣款 → 帖子存入 red_amount/red_remaining
- 带分类导航，发完自动跳转

### 3. 帖子详情与互动

- 点击帖子卡片 → `/post/<id>` 详情页
- 点赞/码住/举报：AJAX 异步提交，按钮即时切换，不跳页
- 评论区：查看评论 + 发表评论
- 显示发布者信息（昵称、学校、等级、经验值）

### 4. 红包分配机制

```
创建红包帖 → 扣发帖者 red_balance → 帖子存 red_amount
           → 评论区用户回复
           → 发帖者选择评论者分配金额
           → 扣帖子 red_remaining → 加接收者 red_balance
           → 记录到 red_distribution
           → red_remaining = 0 → 自动结束帖子
```

- 发帖者也可手动点「结束帖子」
- 每条分配有完整记录可追溯

### 5. 消息系统

| Tab | 内容 | SQL 逻辑 |
|-----|------|---------|
| 回复我的 | 别人在我帖子下的评论 | comments JOIN posts WHERE posts.user_id = me AND comments.user_id != me |
| 收到的赞 | 别人对我帖子的点赞 | likes JOIN posts WHERE posts.user_id = me AND likes.user_id != me |
| 我的点赞 | 我点过赞的帖子 | likes JOIN posts WHERE likes.user_id = me |
| 我的码住 | 我收藏的帖子 | favorites JOIN posts WHERE user_id = me |

### 6. 等级与经验系统

```
小学生(0) → 初中生(200) → 高中生(400) → 大学生(600) → 研究生(800) → 博士(1000)
```

每 200exp 升一级，每级显示对应 emoji 和进度条。

### 7. 浏览去重

`post_views` 表（post_id, user_id 联合主键），同一用户对同一帖子只计一次浏览。

### 8. SPA 单页导航

JavaScript 拦截侧边栏和卡片点击，fetch 加载内容后替换主区域，实现无整页刷新页面切换。事件委托机制，全局绑定一次。

### 9. 亮暗主题切换

CSS 变量双主题方案，`localStorage` 记忆偏好，切换按钮在侧边栏。

### 10. 搜索

集成在浏览页面，支持关键词模糊搜索（标题+内容）和按点赞数筛选。

## 六、部署说明

### 环境要求

- Python 3.8+
- MySQL 8.0+
- 依赖：Flask, pymysql

### 部署步骤

```bash
# 1. 导入数据库
mysql -u root -p < campusmarket_full.sql

# 2. 修改 db.py 中 MySQL 密码

# 3. 安装依赖
pip install flask pymysql

# 4. 启动
python app.py
# 浏览器打开 http://localhost:8501
```
